Run this checklist once per course before you publish. It takes about 15 minutes and catches the vast majority of accessibility regressions.

## Pre-publish accessibility checklist

### Text content
- [ ] All headings use H2–H4 (not bold-as-heading).
- [ ] Every paragraph fits the reading-age target (Hemingway grade ≤ 10).
- [ ] No "click here" links — every link describes its destination.
- [ ] Acronyms expanded on first use (GDC → General Dental Council).

### Images
- [ ] Every image has alt text.
- [ ] Decorative images use `alt=""`.
- [ ] Clinical images include a short description in the body text, not just the alt.
- [ ] Image-only diagrams have a text equivalent in the unit.

### Video
- [ ] Every video has a transcript or accurate captions.
- [ ] Auto-captions have been reviewed (not raw YouTube output).
- [ ] No essential information conveyed only in the audio.

### Problems and XBlocks
- [ ] Every problem can be answered with keyboard only.
- [ ] Image-based problems have a text-based alternative or clear alt text.
- [ ] No time limits, or time limits are extendable on request.

### Colour and contrast
- [ ] No information conveyed by colour alone.
- [ ] No inline colour overrides on text (let the theme handle it).

### Course-wide
- [ ] Course-about page describes the course in plain English.
- [ ] Course overview lists CPD hours and GDC outcomes addressed.
- [ ] Support email is visible: [dental.cpd@liverpool.ac.uk](mailto:dental.cpd@liverpool.ac.uk).
